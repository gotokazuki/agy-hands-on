## プロジェクト共通ルール

このファイルはプロジェクト全体の基本方針を定義する。エージェントはすべてのタスクにおいて以下の内容を遵守すること。

- プロジェクト構成
  - モノレポ構造を維持する（backend, frontend, docs）
  - すべての実行環境は Docker Compose で管理する

- 技術スタック（コア）
  - Java 25 / Spring Boot 4.0
  - React 19 / TypeScript
  - PostgreSQL 18
  - OpenAPI Specification 3.1
  - 認証・認可: JWTベース。詳細は `.agent/rules/auth_mapping.md` を参照。

- 言語設定
  - ソースコード内のコメントは英語を使用する
  - README 等のドキュメントは日本語を使用する
  - 変数名、関数名、クラス名などの識別子は英語を使用する

- エージェントの行動指針
  - 破壊的な変更を伴う操作の前には、必ず実行計画を提示し承認を得ること
  - ターミナルの実行結果を監視し、エラーが発生した場合は即座に原因を解析して修正案を提示すること
